# Action JSON Schema

> KB 每条 entry 的 `interventions` 字段，以及诊断接口 `/diagnose` 响应的 `actions` 数组里，每个元素都遵循这个 schema。
>
> 锁定 v1 · 2026-07-06

## 1. Schema

```json
{
  "label": "<动词+对象, 4-12 字>",
  "do":    "<具体动作, 30-60 字, 用中文>",
  "when":  "now | today | this_week",
  "min":   <int, 预计分钟>,
  "mark":  "<完成的判定, 一行>"
}
```

## 2. 字段细则

| 字段 | 类型 | 必填 | 含义 |
|---|---|---|---|
| `label` | string | ✓ | 动词+对象，4-12 字，例 "5秒起跑"、"Memento Mori 实物提醒" |
| `do` | string | ✓ | 具体步骤，1-2 句，**动作**而非概念。例 "倒数5秒立刻打开文档输入第一句（不限好坏）" |
| `when` | enum | ✓ | `now`（5 分钟内）/ `today`（今天内）/ `this_week`（本周内） |
| `min` | int | ✓ | 预估花费分钟。`0` = 已嵌入日常的瞬间动作（"接受"、"承认"） |
| `mark` | string | ✓ | 完成判定的句子，可量化最好 |

## 3. 命名硬约束

- **每个 KB 标签至少 1 个 `now` 动作**：用户的"救命稻草"是 now
- **每个 KB 标签至少 1 个 `today` 动作**：今天能开工的事
- **每个 KB 标签至少 1 个 `this_week` 动作**：周日复盘时能回头看的
- **default 3 个动作**，可允许 4，最低 1，最高 4
- **`do` 是动词不是概念名**：禁止 "5秒法则"，要 "倒数5秒立刻打开文档输入第一句"
- **`mark` 是判定不是感想**：禁止 "感觉好了一些"，要 "你输出了第一句"

## 4. 对比示例

| ❌ 不合格 | ✅ 合规格 |
|---|---|
| `{label: "5秒法则"}` | `{label: "5秒起跑"}` |
| `{do: "用5秒法则"}` | `{do: "倒数5秒立刻打开文档输入第一句(不限好坏)"}` |
| `{label: "WOOP"}` | `{label: "WOOP四问"}` |
| `{do: "做WOOP"}` | `{do: "写下W(想达成)/O(成功的样子)/O(卡点)/P(应对)"}` |
| `{label: "降低启动成本"}` | `{label: "拆启动动作"}` |
| `{do: "降低动作粒度"}` | `{do: "把'起草段'改成'打开文档'，单一按钮式启动"}` |
| `{mark: "感觉好了一点"}` | `{mark: "你输出了第一句(烂也算赢)"}` |

## 5. K-id 与 actions 数量现状（v1 锁定）

111 entries × 默认 3 个动作 = 333 个结构化动作对象。

| when | 数量 | 占比 | 说明 |
|---|---|---|---|
| `now` | 64 | 19% | 5 分钟内可做 |
| `today` | 146 | 44% | 今天内可做 |
| `this_week` | 123 | 37% | 本周内可做 |

> 占比合理：`now` 留少（保留为"救命稻草"），主力放在 `today` 和 `this_week`。

## 6. 服务端排序逻辑（`top_actions_for` in `app/kb.py`）

```python
# 输入: 一组 K-id + max_n
# 流程:
#   1. 把所有 K-id 的 actions 拉出来放一个 pool
#   2. 按 (when_rank, min, label) 排序:
#      when_rank: now=0, today=1, this_week=2
#      min: 升序
#   3. label 去重（同名动作只保留第一条）
#   4. 截断到 max_n=3
```

**结果**：每条 diagnosis 返回 top-3 actions，其中 `actions[0]` 是用户的"立刻能做的"。

## 7. 不变量（修改时保护）

| 不变量 | 怎么保 |
|---|---|
| 333 个动作对象 | `[scripts/audit_kb.py](../scripts/audit_kb.py)` 不检查这个,但 `refactor_actions.py` 一次性写入保证 |
| `when` 分布大致均衡 | 上面 §5 表里看到现状。增/删 entry 时保持均衡 |
| 所有 K-id 都有非空 `interventions` | audit 检查 |
| `do` 不是纯概念名 | 审稿时人工检查 |
