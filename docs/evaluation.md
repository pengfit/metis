# 测试场景与评估

> 用于评估诊断 prompt / KB 覆盖 / 服务端排序逻辑的 10 段真实情境。
>
> 源位置（之前）：`Knowledge Base/_eval/scenarios.md`

## 1. 用途

每次改动以下东西时，**至少跑这 10 段**：
- 25 个 v1 标签（扩/缩）
- Prompt（措辞、约束）
- KB 的某条目（增/删/改）
- 服务端排序逻辑（`top_actions_for`）

## 2. 评估流程

### 手动（人工）

```bash
# 1) 服务起来
python3 -m uvicorn app.main:app --port 8000 &

# 2) 对每个场景 S0X 复制 input, 浏览器点按钮 或 curl:
curl -X POST http://localhost:8000/diagnose \
  -H 'Content-Type: application/json' \
  -d "$(jq -n --arg s '$(<scenario.txt)' '{input: $s}')"
# 或者直接浏览器, 复制/粘贴进 textarea

# 3) 检查输出:
#    - tag 是否在 25 个 v1 内?
#    - confidence 在合理范围?
#    - evidence 引用了用户原话?
#    - k_ids 全部真实存在?
#    - actions 是不是可执行的具体动作?
#    - actions[0] 是不是"立刻能做"?
```

### 自动化（`scripts/test_e2e.py`）

`scripts/test_e2e.py` 用 `fastapi.testclient.TestClient` 跑 8 个核心场景，端到端验证 schema。可以扩。

## 3. 10 段测试场景

### S01 · 拖延-完美
> 最近有个重要提案拖了两周没动。资料都查齐了，就是写不出第一段。开始写就觉得"开头不够惊艳"，改来改去几小时没成型。

**期望命中：** `拖延` / `完美主义`
**应给出 K-id：** K035 / K036 / K060

### S02 · 疲惫-空虚
> 周末睡 12 小时起来还是累。上班像被推着走。也说不上讨厌工作，就是看不出"在为谁做"。回到家手机刷两小时就睡着了，醒来还是没充电的感觉。

**期望命中：** `疲惫` / `空虚`
**应给出 K-id：** K051 / K057 / K008 / K038；意义方向可考虑 K031

### S03 · 焦虑-反刍
> 后天要面谈升职的事。今晚开始刷各种可能的尴尬场景，洗澡时也想，越想越觉得会搞砸，胃也开始疼。理性知道"想一百遍也没用"，但停不下来。

**期望命中：** `焦虑` / `反刍` / `极端思维`
**应给出 K-id：** K019 / K020 / K032 / K078

### S04 · 身份迷茫
> 毕业两年换了 4 份工作，每次都热情几个月就冷淡下来。被朋友说"你好像不知道自己想要什么"。我也觉得好像确实——但我每次开始时是真的喜欢啊，怎么就腻了？

**期望命中：** `身份迷茫` / `信念冲突`
**应给出 K-id：** K030 / K014 / K045 / K062

### S05 · 关系摩擦
> 最近跟伴侣老为小事吵。他昨天问"你怎么没回消息"，我立刻发火。后来看，他只是关心，但我第一反应是防御。好几次了，每次都这样过完才后悔。

**期望命中：** `关系摩擦` / `信念冲突` / `易怒`
**应给出 K-id：** K066 / K099 / K003 / K073

### S06 · 被动等待
> 自由职业三年。每个月都觉得下个月会爆发，结果什么都没主动做。客户找上门才接，没人找就刷手机过去。方向是有的——心理咨询执业，只是没迈出去报名考试。

**期望命中：** `被动等待` / `复利失灵` / `信念冲突`
**应给出 K-id：** K050 / K002 / K056 / K009

### S07 · 转型期
> 公司被收购。我的职位还在，但汇报线变了，团队散了一半。新老板不认识我。这种"还在但什么都不一样"的状态比"被裁"还磨人——我没决定权，但又要我当一面。

**期望命中：** `转型期` / `疲惫` / `复利失灵`
**应给出 K-id：** K023 / K064 / K092 / K100

### S08 · 信念冲突（拖延的另一种脸）
> 明知道要规律作息，但每天晚上 11 点就开始刷手机到 2 点。第二天自责，第二晚又来。理性知道这在伤害自己，但睡前那一刻就是觉得"今天已经废了，不如就这么废了"。

**期望命中：** `信念冲突` / `回避` / `证据失效`
**应给出 K-id：** K014 / K018 / K035 / K069

### S09 · 赛道疑问
> 在公司做了 5 年会计，待遇不错，但总觉得自己像金丝笼里的。35 岁，最近在想要不要转数据分析，从头开始。伴侣说我"又来了"——其实也没错，三年前也想过转。

**期望命中：** `赛道疑问` / `信念冲突` / `转型期`
**应给出 K-id：** K063 / K055 / K094 / K072；身份方向 K062 / K030

### S10 · 死亡焦虑
> 这一年好几个朋友家里出事了。四十五十就走。我爸最近体检查出血脂高。突然意识到时间不是无限的，我好像什么都没"留下来"。夜里醒过来想这事，白天装作没事。

**期望命中：** `死亡焦虑` / `意义缺失` / `回避`
**应给出 K-id：** K110 / K031 / K039；情绪方向 K061 / K005

## 5. 评估 checklist

每次跑完 10 场景，打分：

| 检查项 | 通过标准 |
|---|---|
| 标签命中正确 | 期望/实际 tag 一致率 ≥ 80% |
| 没有幻觉标签 | outputs 里 0 个 v1 外的标签 |
| 没有幻觉 K-id | outputs 里 0 个 KB 不存在的 K-id |
| evidence 引用原文 | 每个 evidence 都从原话短语里抽 |
| confidence 合理 | 不全 0.9+，不全 0.3- |
| actions[0] 是 now | 一致 80%+ |
| actions 是动作动词 | 100% 真实动作（没有"5秒法则"等纯概念） |
| 响应时间 | < 5s (mock) / < 30s (real LLM) |
| **kb_hits 包含期望 K-id** ★ v1.2 | 期望 K-id 出现在 `kb_hits` top-10 中 ≥ 60% |
| **time_context 与服务器时间一致** ★ v1.2 | hour 字段与服务器本地时间匹配 |
| **深夜过滤生效** ★ v1.2 | 22:00-06:00 调诊断, actions 不含"剧烈运动""联系朋友""出门散步" |

## 6. 失败模式如何记录

LLM 答错时，先记下：
1. 输入是什么
2. LLM 输出是什么（完整 JSON）
3. 你期望它应该输出什么
4. 为什么偏差

积累 10 条失败案例后，做一次"prompt 调优周"。

## 7. v1.2 增量评估

### KB 检索召回率评估

`kb_hits` 字段反映服务端 hybrid 检索命中的 top-K K-id + 融合分数。可独立评估检索层质量：

```python
# 手工验证脚本
import requests
expected_kids_per_scenario = {
    'S01': ['K035','K036'],
    'S03': ['K019','K020','K032'],
    'S08': ['K014','K005'],
    'S10': ['K110','K031','K039'],
}

for sid, expected in expected_kids_per_scenario.items():
    r = requests.post('http://localhost:8000/diagnose',
        json={'input': USER_INPUTS[sid]})
    hit_kids = {h['kid'] for h in r.json()['kb_hits'][:10]}
    recall = len(set(expected) & hit_kids) / len(expected)
    print(f'{sid}: recall@10 = {recall:.0%}, hits = {hit_kids & set(expected)}')
```

参考结果 (v1.2 验收):
- S01: recall@10 = 100% (K035/K036 全中)
- S03: recall@10 = 33% (K019 中, K020/K032 漏——抽象概念名与症状描述鸿沟)
- S08: recall@10 = 0% (K014/K005 全漏——纯抽象概念)
- S10: recall@10 = 67% (K110/K039 中, K031 漏)

**结论**: hybrid 检索对"症状→具体概念"映射有效 (S01/S03/S10), 对"症状→抽象概念"映射仍是天花板 (S08)。**LLM 在 KB context 候选里能进一步选对**（4 场景标签命中 100%），所以 KB context 是参考不是束缚。
